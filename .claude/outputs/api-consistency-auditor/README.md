# API Consistency Auditor - Output Directory

This directory contains outputs from the API Consistency Auditor agent.

## Purpose

The api-consistency-auditor agent writes all code reviews, violation reports, and audit results to this directory following the subagent markdown output pattern.

## Output Types

### Code Reviews
**Naming**: `{date}-{filename}-review.md`
**Example**: `2025-12-15-catalog-review.md`
**Content**: Detailed violation analysis with fix suggestions

### Baseline Audits
**Naming**: `{date}-baseline-audit.md`
**Example**: `2025-12-15-baseline-audit.md`
**Content**: Comprehensive audit of current codebase state

### Fix Suggestions
**Naming**: `{date}-fix-suggestions-{scope}.md`
**Example**: `2025-12-15-fix-suggestions-catalog.md`
**Content**: Before/after code examples for fixes

### Batch Reports
**Naming**: `{date}-batch-audit-{description}.md`
**Example**: `2025-12-15-batch-audit-recent-additions.md`
**Content**: Multi-file audit results

## Standard Format

All outputs follow this structure:

```markdown
# API Consistency {Type} - {Scope}

**Date**: {YYYY-MM-DD}
**Agent**: api-consistency-auditor
**Files**: {filepath(s)}

## Summary
- Total violations: X
- Critical: Y
- Warnings: Z

## Violations Found

### {File 1}
#### Rule 1: {Rule Name}
- **Line**: XX
- **Violation**: {Description}
- **Current**: {Code snippet}
- **Fix**: {Fix suggestion}

[... all violations ...]

## Fix Suggestions

### {File} Refactor
**Before**:
```python
# Current code
```

**After**:
```python
# Fixed code
```

## Next Steps
1. {Action item}
2. {Action item}

---
*Generated by api-consistency-auditor agent*
```

## Lifecycle

**Active**: Outputs written during current work
**Archive**: Move to `.old/` when superseded or task complete
**Cleanup**: User reviews `.old/` for deletion candidates
