## How to Use This File

### Task ID Format
Use short, descriptive IDs: `feat-001`, `fix-002`, `docs-003`, `nb-004`

### Task Lifecycle
```
READY → IN PROGRESS → COMPLETED → ARCHIVED (.old/tasks/)
```

### Moving Tasks
1. When starting: Move from Ready → In Progress
2. When blocked: Move to Blocked, note the blocker
3. When done: Move to Completed, check the box
4. Periodically: Archive old completed tasks to `.old/tasks/`

### Adding New Tasks
Add to Ready section if no dependencies, Blocked if waiting on something.
